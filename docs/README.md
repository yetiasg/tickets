# Docs

This directory stores project documentation that should stay aligned with the implementation.

Expected content includes:

1. module documentation,
2. entity documentation,
3. business solution documentation,
4. architecture descriptions,
5. diagram sources and rendered diagrams,
6. Bruno API collections.

Suggested structure:

1. `docs/modules/`
2. `docs/entities/`
3. `docs/business-solutions/`
4. `docs/bruno/`
5. `docs/infrastructure-architecture.mmd`

Documentation should explain not only what exists, but also why it exists.
